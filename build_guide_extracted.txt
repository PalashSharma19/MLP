Project Overview
You will build a malware detection system that:
Converts binary executable files (.exe, .dll, .bat) into grayscale images
Trains a CNN (ResNet-18 fine-tuned) on the MalImg dataset
Trains traditional ML models (Random Forest + SVM) on HOG/statistical features extracted from the same images
Compares CNN vs ML model performance with a rich evaluation report
Produces a clean comparison dashboard saved as both CSV and PNG charts

Why These Features?
Traditional ML models cannot process raw 224x224 pixels (50,176 dims) efficiently. Instead we extract compact, meaningful descriptors:
Feature
What it captures
Dimension
HOG (Histogram of Oriented Gradients)
Edge direction patterns, shape and texture structure in image regions
~1764
Mean intensity
Overall brightness level of the image
1
Std deviation
Contrast / dynamic range of pixel intensities
1
Entropy
Randomness / complexity of pixel distribution
1
Byte distribution uniformity
How spread out the byte values are (packed malware looks uniform)
1
Zero-byte ratio
Proportion of null bytes (common in padding sections)
1

FUNCTION: extract_features_single(image_path)
Load image with PIL in mode L (grayscale). Convert to numpy uint8 array.
Extract HOG: skimage.feature.hog(img, orientations=9, pixels_per_cell=(8,8),
cells_per_block=(2,2), feature_vector=True)
Resize image to 128x128 first before HOG (use cv2.resize).
Compute: mean, std, shannon_entropy, zero_ratio (pixels==0 / total), byte_uniformity
(byte_uniformity = 1 - (std / 128.0), clipped to [0,1]).
Return np.concatenate([hog_features, [mean, std, entropy, zero_ratio, byte_uniformity]])
FUNCTION: extract_features_from_csv(csv_path, cache_path=None)

If cache_path provided and file does not exist, save result to cache after extraction.

Why CNN Almost Always Wins
CNN learns hierarchical features directly from pixels, while ML models rely on pre-defined HOG descriptors. HOG is designed for natural images and may miss malware-specific patterns. CNN adapts its filters to the domain during fine-tuning. However, the gap is smaller than you'd expect — HOG captures edge patterns that are genuinely discriminative for malware families, and this narrow gap is academically interesting to discuss.
When ML Models Win
On very small datasets or heavily augmented data, Random Forest can outperform a fine-tuned CNN. Also, SVM with RBF kernel can match CNN accuracy on well-structured texture classification tasks like this. If your training data is < 500 samples per class, consider this as a finding.
ACADEMIC ANGLE FOR REPORT
CNN advantage: No manual feature engineering, end-to-end learning, captures non-linear texture hierarchies.
ML advantage: Interpretable (RF feature importances), faster training, no GPU required, less overfitting risk on small data.
Key research question: Does the CNN’s black-box feature extraction justify its complexity over interpretable HOG+ML?
Answer from results: Yes for accuracy, but not necessarily for interpretability or deployment cost.

Fix

Add --skip-ml flag and train SVM separately on a subset
